Project proposals due next week, April 14

By the second week of class, students should submit a written proposal for their class project

The proposal should include the following :

  • Who is on the project team
  • One paragraph describing the finished product
  • Which version control system is to be used (e.g. github or a private VCS)
  • A brief high level plan of attack. This could include architecture, test plans, development strategies, QA, ..

Please email your proposal to the instructors at joseph.sheedy@gmail.com and PythonCHB@gmail.com

If working in a group, only one submission is necessary per group. Cc: all members of the group.
